# React (Vite) Deployment Guide for GitHub Pages

This guide explains how this project is configured to deploy to **GitHub Pages** (using a custom domain like `researchnest.me`) and connect to a hosted backend. You can use these exact steps to set up and deploy any React Vite project.

---

## 1. How the System Works (The Core Concept)

When deploying a React/Vite app to GitHub Pages, the project uses two branches:
1. **`main` (or `master`) branch**: Stores your **source code** (`.jsx`, `.js`, CSS, components). Browsers cannot read these files directly.
2. **`gh-pages` branch**: Stores the **compiled web files** (HTML, CSS, compiled JS) generated by Vite. This is what the browser loads to display the website.

Whenever you want to update the live website, you must **compile (build)** your source code and push the new build files to the `gh-pages` branch. 

---

## 2. Step-by-Step Setup Guide

If you are setting this up on a new project, follow these steps:

### Step 1: Install the Deployment Tool
In your terminal, navigate to your frontend client folder and install the `gh-pages` package as a developer dependency:
```bash
npm install gh-pages --save-dev
```

### Step 2: Configure Vite for Subfolders & Custom Domains
Open `vite.config.js` and add `base: './'` (relative base path). This ensures your compiled HTML file correctly references style sheets and JS scripts, whether the site is hosted in a subfolder (`/ServerNear/`) or at the root of a domain.

```javascript
// vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  base: './', // <--- Crucial for path resolutions
})
```

### Step 3: Add Deploy Scripts to `package.json`
Open your client's `package.json` file and add the `predeploy` and `deploy` commands into the `"scripts"` block:

```json
"scripts": {
  "dev": "vite",
  "build": "vite build",
  "predeploy": "npm run build",
  "deploy": "gh-pages -d dist"
}
```
* **`predeploy`**: Automatically triggers `npm run build` to compile the app into a folder named `dist`.
* **`deploy`**: Uploads the compiled `dist` folder directly to the `gh-pages` branch on GitHub.

### Step 4: Use HashRouter instead of BrowserRouter (For React Router)
GitHub Pages does not support server-side routing redirects. If a user visits `yoursite.com/services` directly or refreshes the page, GitHub Pages returns a **404 Not Found** error.

To avoid this, import and wrap your app in `HashRouter` instead of `BrowserRouter`:

```javascript
// App.jsx or main.jsx
import { HashRouter as Router, Routes, Route } from 'react-router-dom';

function App() {
  return (
    <Router>
      <Routes>
        <Route path="/" element={<Home />} />
        {/* other routes */}
      </Routes>
    </Router>
  );
}
```
*This appends a `#` to your URLs (e.g. `researchnest.me/ServerNear/#/services`), telling the browser not to request the `/services` folder from GitHub Pages' server, keeping everything functional inside React.*

### Step 5: Don't Hardcode Image Paths (e.g., Logo)
Avoid absolute image paths like `<img src="/logo.svg" />`, because the browser will look for the image at the root of the domain (`yoursite.com/logo.svg`) and miss the subdirectory.

Instead, import the image at the top of your React component so Vite can package and reference it dynamically:

```jsx
// 1. Move your image to src/assets
import logo from '../assets/logo.svg';

// 2. Reference the variable in JSX
<img src={logo} alt="Logo" />
```

### Step 6: Set Dynamic API Base URLs
Set your API URLs using environment variables so that your code automatically knows when to talk to `localhost` vs your live cloud backend (like Render).

Create a file named `.env.production` inside the client folder:
```env
VITE_API_URL=https://your-backend-on-render.com/api
```

In your Axios configuration or API utility:
```javascript
const API = axios.create({
  baseURL: import.meta.env.VITE_API_URL || 'http://localhost:5000/api',
});
```

## 3. How to Deploy Edits

Once the setup above is complete, you can update your live website anytime by running:

1. **Edit** your code in VS Code.
2. Open your terminal in the `client` directory and run:
   ```bash
   npm run deploy
   ```
3. Wait **1 minute** for GitHub Pages to update.
4. Go to your site and perform a **Hard Refresh** to clear your browser's memory cache:
   * **Windows/Linux**: Press `Ctrl` + `F5` (or `Ctrl` + `Shift` + `R`).
   * **Mac**: Press `Cmd` + `Shift` + `R`.

---

## 4. Backend CORS Configuration (For Full Integration)
If your frontend is live, your hosted backend must give it permission to request data. Add your live domain to the backend's CORS whitelist:

```javascript
// server.js (on your Express backend)
app.use(cors({
  origin: ['http://localhost:5173', 'https://researchnest.me'], // Add frontend domain
  credentials: true
}));
```
